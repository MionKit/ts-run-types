The JSON columns show four encoder strategies. clone (the default) builds a fresh object from the declared shape and drops undeclared keys. mutate transforms the value in place and keeps extras. direct is a single pass. compact writes each object's declared properties as a positional array with no key names on the wire, then rebuilds the object on decode.

In many real cases compact is the better choice than binary. Both compact and binary drop the repeated field names, so their payloads are close in size. The difference is speed: compact keeps the heavy work in native `JSON.stringify` and `JSON.parse` (which the engine runs in C++), while the binary codec reads and writes bytes one at a time in JavaScript. So compact encodes and decodes much faster while staying nearly as small, which usually wins the end to end round trip (small payload plus native speed parsing) for typical RPC and network traffic. Binary stays the choice for numeric or byte heavy payloads, where its tighter packing of numbers dominates.
