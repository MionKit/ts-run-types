# Unified src-derived friendly reconcile — one type, precise `$errors`, translations from src

> **Status: SHIPPED — implemented as specced on the
> `claude/friendlytypes-i18n-l8e2y3` branch (PR #166).** Supersedes the
> friendly-mirror-derived translation driver that PR shipped first
> ([friendly-type-i18n.md](./friendly-type-i18n.md) §Reconcile — see its
> deviation #11); the final architecture merged in one piece. Implementation
> notes vs the plan: type names for translation targets come from the
> `FriendlyType<T>` annotation (not the `@rtType` marker name), and
> `ResolveTypeRaw` runs once per type with only `EmitClosure` per locale.
> Follow-up shipped in the same PR: every meta key this document spells `$X`
> ships as `rt$X` — the reserved-prefix rename
> ([enrich-meta-key-prefix.md](./enrich-meta-key-prefix.md)).

## Motivation (decided in design dialogue, 2026-07-03)

1. **`generated/` must never be an input to generating another file.** Today a
   translation's desired side is the friendly SOURCE MIRROR — a generated file
   feeding generation. That makes the friendly tree load-bearing, creates the
   two-hop staleness cascade (src → friendly → translations), and blocks ever
   treating the generated dirs as watch-ignored/write-only. (A reconcile still
   reads its OWN target file — value preservation is read-modify-write by
   definition — but no file under `generated/` may drive another.)
2. **One reconcile.** Every friendly-family file (the source-language mirror
   AND each locale file) syncs from the src TYPE with the same
   EmitClosure-driven, value-preserving reconcile, parameterized per output.
3. Verified bonus: the `$errors` descent is currently gated `if ctx.translate`
   (merge.go) — the ordinary friendly reconcile never scaffolds a newly added
   constraint key into an existing node. Unification un-gates the descent for
   every mirror and fixes that gap as a side effect.

## Decisions (all DECIDED)

1. **One type.** `Translation<T>` is deleted; `FriendlyType<T>` annotates every
   friendly-family const (`pl_friendlyUser: FriendlyType<User>`). The
   `FriendlyText` rename is a separate follow-up
   ([friendly-text-rename.md](../todos/friendly-text-rename.md)).
2. **Function-form `$errors` is REMOVED entirely** (`(failed) => string`): it
   is opaque to translation, reconcile and the checker. Only data survives.
   `FailedConstraint` / `FailedConstraints` exports go with it.
3. **Param-precise `$errors` typing.** The field's format params drive the
   record type: one REQUIRED key per failable param (blank `''` = "no custom
   message" — the opt-out; deleting a key just re-scaffolds it), count-bearing
   keys accept `TemplateLeaf` (plural), everything else `FriendlyTemplate`,
   NO index signature (unknown key = excess-property error → FT003 moves into
   the IDE). Prototype measured on a 4-field interface: 171 → 482
   instantiations, 0.05s → 0.13s — acceptable because `FriendlyType`
   instantiations are TERMINAL (declared once per mirror, consumed by
   renderers; they never propagate through consumer code the way `DataOnly`
   does). Re-baseline the enrichHarness compile-budget test.
4. **`$default` is kept, MUTUALLY EXCLUSIVE with per-constraint keys** —
   `{$default: '…'}` XOR the per-constraint record, enforced by the TS union
   AND a Go checker rule (belt-and-braces, new FT0xx Error). With arrows gone
   it is the only single-message-per-field mechanism and the only translatable
   catch-all (the renderer's hardcoded `"<label> is invalid"` is English).
5. **`friendlyErrors` config knob** (tsconfig plugin entry, top level):
   `"perConstraint"` (default) | `"default"` — which `$errors` mode `gen`
   scaffolds for a NEW node. Once a node exists, its authored mode is
   author-owned and the reconcile follows it (a `$default`-only node is
   skipped by the descent entirely; a per-constraint node gets the normal
   descent).
6. **Orphan rule for the `$errors` descent:** orphan ONLY keys attributable to
   the type — recognized constraint names (the catalog below) the field no
   longer declares. Unknown keys are author-owned and untouched (TS flags
   typos as excess properties anyway).
7. **Translations derive from src.** A locale file is generated by the SAME
   EmitClosure walk as the friendly mirror, with four parameters: const prefix
   (`pl_friendly*`), output path (`<i18nDir>/<locale>/<rel>.ts`), plural arms
   from the TARGET locale's CLDR set, sibling value-refs renamed + re-pathed to
   locale siblings. The breadcrumb is the normal src type import; the
   `@rtI18n <locale> from '<mirror>'` marker DIES (path + const prefix carry
   the locale). Orphan oracle = the one `SourceDeclaresType`. "Never copy
   source text" becomes true by construction (the type has no strings).
8. **`gen --translate` builds the checker Program** (same cost as `gen`); one
   Program amortizes across all locales of a run. TR003 ("out of date")
   becomes "a src-driven reconcile would change the file". Follow-up (not in
   scope): a one-pass `gen --update` that syncs friendly + all configured
   locales together.

## The two sync points (TS ↔ Go — keep aligned or the design drifts)

- **`NonFailingParams`** (params that never fail; never `$errors` keys):
  `isCurrency`, `mockSamples`, `trim`, `lowercase`, `uppercase`, `capitalize`,
  `replace`, `replaceAll`. TS: `friendlyType.ts`; Go: `nonFailingParams` in
  [internal/enrichment/enrich.go](../../internal/enrichment/enrich.go) (extends the
  existing `presentationParams`; also cleans up Go's current "transformer
  params may appear too, author prunes" scaffold fuzziness).
- **`CountBearingKeys`**: `minLength`, `maxLength`, `min`, `max`, `lt`, `gt` —
  TS union mirrors Go's `CountBearing`
  ([internal/enrichment/classify.go](../../internal/enrichment/classify.go)).

## Implementation plan

- **A. TS type + runtime** — `friendlyType.ts`: `ConstraintTemplates<P>` /
  `DefaultOnlyTemplates` / `BareTemplates` / generic `ErrorTemplates<F>` +
  `FriendlyMeta<F>`; leaf arm threads `T`; delete `Translation`,
  `FailedConstraint(s)`, the function arm. `createFriendlyText.ts`: drop the
  function-form branch (the existing `key ?? $default` lookup already serves
  both modes); loose `ErrorTemplatesRuntime` view. `index.ts` exports updated.
- **B. Go scaffold + checker** — `friendlyErrors` config threaded
  tsconfig → enrichConfig → EmitOptions; scaffold emits `{$default: ''}` or
  the per-constraint record for NEW nodes; `nonFailingParams` excluded from
  scaffold keys and FT003's allowed set (done for `isCurrency`; extend to
  transformers); new FT0xx: `$default` beside any other key → Error.
- **C. Reconcile unification** — un-gate `mergeErrorsNode` (all mirrors);
  mode-aware skip for `$default`-only nodes; constraint-catalog orphan rule;
  plural-arm locale-ownership rules unchanged.
- **D. Src-derived translate driver** — rewrite
  [cmd/ts-runtypes/enrich_translate.go](../../cmd/ts-runtypes/enrich_translate.go)
  to build the Program and reuse the normal gen pipeline with the four locale
  parameters; target discovery stays "sources that have a friendly mirror"
  (derivable from path math alone). DELETE: `translate.go`
  (translationBlanker/TranslationConsts), `SourceMirrorDeclaresConst`,
  `TranslateSpec.SourceMirrorPath` plumbing, the `@rtI18n from` emission, the
  `Translation` wrapper in `friendlyWrapper`.
- **E. Tests** — mirror/merge tests for the un-gated descent + mode skip +
  catalog orphans; translate tests rewritten src-driven (Go + the JS e2e);
  createFriendlyText/I18n runtime tests re-typed with real format brands (precise
  typing makes plain-`string` fixtures with `minLength` keys illegal — that's
  the feature); compile-budget re-baseline + new precise-typing cases
  (missing key / unknown key / plural-on-non-count-bearing / `$default` mix
  all `@ts-expect-error`); i18n fuzzer harness reworked src-edit-driven (the
  type-mod fuzzer pattern), oracles carried over.
- **F. Docs** — website i18n + friendly pages, AI_ENRICHMENT, ARCHITECTURE,
  skills ×3 (both copies), configuration page (`friendlyErrors`), the
  friendly-type-i18n done-spec (deviations #11+), PR #166 body; then move THIS
  file to `docs/done/`.

## Acceptance

- No file under `generated/` is read to produce another generated file; the
  friendly mirror and every locale file reconcile from src with one driver.
- `FriendlyType<T>` is the only public map type; arrows and `Translation<T>`
  are gone; precise `$errors` typing catches missing/unknown/mis-pluralized
  keys and `$default` mixing in the IDE; the Go checker mirrors those as
  FT0xx for non-TS contexts.
- `friendlyErrors` picks the first-scaffold mode; authored mode is preserved
  thereafter; `$default`-only nodes are never descended.
- `gen --translate <locale|all> [--update|--prune]` + `check --translate`
  (TR001–TR004) behave as before from the user's view, now Program-driven;
  full Go + JS suites and all three fuzzers green.
