# A `go-generated/` directory per package for the Go→TS mirrors

Status: **DONE** (owner decision 2026-07-18). Supersedes the shared-package proposal
this doc originally explored — see [The decision](#the-decision) for why the shared
package was rejected and this scoped variant chosen instead.

## The request (original)

> Move all autogenerated files from Go to TS into a single new package that every other
> package consumes (devtools and core included), with PR linting to keep it in sync with
> the Go source; website autogen keeps its current behaviour.

## The analysis (why a single shared package was rejected)

Inventory of what the Go side generates for the JS workspace (registry: the `CODEGEN`
map in [`scripts/rt.mjs`](../../scripts/rt.mjs)):

| Target | Committed output | Package | Consumer | Direction |
| --- | --- | --- | --- | --- |
| `constants` | `runtypes-constants.generated.ts` | devtools | devtools only | Go→TS |
| `kind` | `runTypeKind.ts` | core | core (public API) + mocking | Go→TS |
| `kind` | `reflectionKind.generated.ts` | devtools | devtools only | Go→TS |
| `fnhashes` | `fnHashes.generated.ts` | core | core only | Go→TS |
| `diag` | `diagnosticCatalog.generated.ts` | devtools | devtools only | Go→TS |
| `diag` | `diagnostics-catalog.json` | website | website | Go→JSON |
| `builtinpurefns` | `table.generated.go` | Go | resolver | **TS→Go (reverse)** |

Three findings killed the single-shared-package idea:

1. **The sync guarantee already exists.** `pnpm rtx core codegen all --check` (regenerate
   + `git diff`) runs on every PR ([`ci.yml`](../../.github/workflows/ci.yml) +
   [`release-gate.yml`](../../.github/workflows/release-gate.yml)), plus Go tests
   (`TestRunTypeKindFileInSync`, `TestFnHashesFileInSync`). Consolidation added nothing here.
2. **A shared package breaks `@ts-runtypes/core`'s marketed zero-runtime-dependency
   property.** Both FE packages build with plain `tsc` (no bundler, by design), so a
   cross-package import stays a literal runtime dep in the published dist.
3. **Only the kind discriminators are genuinely dual-consumed** (and even those ship as
   two deliberately different shapes). The other four files are single-consumer and
   already in their natural home.

`builtinpurefns` (TS→Go) and the website JSON generator were out of scope for a "move to
a TS package" framing anyway.

## The decision

Keep the current split and functionality (no shared package, no dedup, no API change).
Instead, give each package a dedicated **`go-generated/` directory** that holds **only**
generated files, with a README, so the generated surface is obvious and the
generated/manual boundary is explicit. This captures the discoverability win the request
was really after, at zero risk to the zero-dep + bundler-free properties.

## What shipped

**Relocated every purely-generated file into a `go-generated/` directory** (kept under
`src/` so `tsc`'s `rootDir: src` / `include: [src]` compile them with no config change;
the dist mirrors the layout at `dist/go-generated/`):

- `packages/ts-runtypes/src/go-generated/` — `runTypeKind.ts`, `fnHashes.generated.ts`
- `packages/ts-runtypes-devtools/src/go-generated/` — `runtypes-constants.generated.ts`,
  `reflectionKind.generated.ts`, `diagnosticCatalog.generated.ts`
- `container/website/app/components/content/go-generated/` — `diagnostics-catalog.json`
  (placed next to its only consumer, `DiagnosticCatalog.vue`)

**Added a README** to each of the three `go-generated/` directories: what's generated,
which generator + Go source produces it, how to regenerate, how the drift gate keeps it in
sync, and the **exclusively-generated invariant** — a file in `go-generated/` contains only
generated code; hand-written wrappers (e.g. `diagnosticCatalog.ts`) live in the parent
`src/` and import from `go-generated/`.

**Retargeted the generators + registry** to the new paths:

- Go generator output paths: `cmd/gen-run-type-kind/gen.go` (both `*OutputPath()` funcs +
  the emitted header cross-references), `cmd/gen-fn-hashes/gen.go`, `cmd/gen-ts-constants/main.go`.
- `scripts/core/gen-diagnostics-catalog.mjs` (`generatedTsPath` + `websiteJsonPath`).
- The `CODEGEN` registry in `scripts/rt.mjs` (`outputs` / `fmt` / `stdoutTo`).
- Doc-comment path references in the `cmd/gen-*` sources + `gen_test.go`.

**Updated every importer** — core `index.ts` / `fnHash.ts` / `mocking/*` + tests; devtools
`protocol.ts` / `unplugin.ts` / `eslint/prefilter.ts` / `diagnosticCatalog.ts` + tests; the
website Vue component; and `scripts/website/suite-data/export-validation.mjs` (reads the
devtools **dist** path, now `dist/go-generated/`).

**Docs:** fixed the two `docs/ARCHITECTURE.md` links that pointed at the old paths.

## Sync enforcement (unchanged mechanism, follows the files)

No new lint machinery was needed. `pnpm rtx core codegen all --check` diffs the registry's
`outputs`, so it followed the paths to `go-generated/` for free; the Go `*FileInSync` tests
read via the updated `*OutputPath()` funcs. Both gates already run on every PR.

## Verification

- `go -C ts-go-runtypes test ./cmd/... ./internal/...` — green (incl. the `*FileInSync`
  sync tests); `table.generated.go` byte-identical after regenerate.
- `pnpm rtx core codegen all --check` — clean.
- `pnpm run check:builds` (resolver + both dists), `pnpm run typecheck`, `pnpm run lint`,
  `pnpm run check-format` — all green.
- JS suite (`vitest run --exclude '**/test/fuzz/**'`) — 194 files / 7965 tests passed;
  the cloning + value fuzz suites (which import the moved `runTypeKind` helpers) — green.

## Notes / deliberate choices

- **`go-generated/` sits under `src/`, not the package root.** `rootDir: src` means a
  package-root dir would force `rootDir: "."` and shift every dist path (breaking the
  exports map). `src/go-generated/` is a dedicated, clearly-named, README-documented
  directory that compiles and publishes unchanged.
- **Basenames were preserved** (`runTypeKind.ts` keeps no `.generated` suffix) to keep the
  diff path-only; the directory now carries the "generated" signal.
- **`builtinpurefns/table.generated.go` was not moved** — it is a Go file generated from TS
  (reverse direction), consumed by the resolver.
