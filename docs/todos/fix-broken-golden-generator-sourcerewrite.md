# `sourcerewrite/testdata/gen_golden.mjs` is broken — golden fixtures can't be regenerated

## Intent

The `TestApply_Golden` fixtures under [ts-go-runtypes/internal/compiler/sourcerewrite/testdata/](../../ts-go-runtypes/internal/compiler/sourcerewrite/testdata/) (`*.json`) can no longer be regenerated by their generator. `gen_golden.mjs` is **doubly stale** — both breakages predate the change that surfaced them (the `createX` → `createXFn` rename PR, which had to work around it):

1. **Stale import path** after the Go-tree relocation (`99e7b177`): the generator imports via `../../../../packages/...`, which resolves to `ts-go-runtypes/packages/` (nonexistent). It needs `../../../../../packages/...` (5 levels up to the repo root). The "run from repo root" header comment is also stale (`internal/…` → `ts-go-runtypes/internal/…`).
2. **Dead dependency (the real blocker):** it imports `{rewrite}` from `packages/ts-runtypes-devtools/dist/rewrite.js`, but that JS rewrite oracle was **deleted** in `7031676c` ("thin Vite wrapper — Go owns the transform via OpTransform"). The function `gen_golden.mjs` drives no longer exists in the dist.

Impact: the golden fixtures are effectively frozen. Any future change to the rewrite / source-map logic can't refresh them, and a change that touches the synthetic fixture source (identifier text) breaks the baked UTF-8 byte offsets with no regeneration path. The `createXFn` PR handled this by **leaving the `testdata/` fixtures at their committed synthetic names** (the callee identifier is immaterial to the byte-offset/sourcemap mechanics under test) and filing this.

## Direction

Rebuild the generator to drive the **current** transform oracle instead of the deleted JS `rewrite()`. Since Go now owns the transform, the natural oracle is the Go binary's `OpTransform` in `'go'` wire mode (returns whole `{code, map}`): spawn `bin/ts-runtypes` (or add a small `ts-go-runtypes/cmd/gen-golden` helper) and feed each case's `{file, code, sites, replacements}`, capturing `expectedCode` / `expectedMap`. Fix the import path + header comment staleness at the same time. The implementer decides whether spawning the prebuilt binary or a dedicated `cmd/` helper is cleaner, and works out the exact `OpTransform` request shape (see the `transformMode: 'go'` path and `protocol` package).

Verify by regenerating all fixtures, confirming `go -C ts-go-runtypes test ./internal/compiler/sourcerewrite/...` stays green, then perturbing one case to confirm the generator actually recomputes offsets (not just rewrites bytes).

## Done when

- `gen_golden.mjs` (or its replacement) runs from the repo root and regenerates every `testdata/*.json` against the current transform, with `sourcerewrite` tests green.
- The stale import path, stale header comment, and dead `dist/rewrite.js` dependency are all gone.
- Optionally: the regenerated fixtures adopt the real API names (`createValidateFn`, …) now that they can be refreshed, replacing the committed synthetic `createValidate` placeholders.
