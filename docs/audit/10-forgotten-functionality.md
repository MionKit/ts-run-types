## 4. Intentional deviations (by design)

1. **No runtime RunType class hierarchy.** The single biggest structural consequence: mion's `RunType` is a live class with methods + a deepkit `Type` back-link; ts-go's is an inert JSON node in `runTypesCache`. This is *why* `lib/guards.ts`, the `RunType` *methods* (¬ß3 #5), `MockJitCompiler`, `createJitFunction`, and `dkProxy` are dropped or relocated ‚Äî they all presuppose runtime reflection objects. The AOT model resolves all of that in Go and ships data + emitted code. (overview ¬ß"Why the architectures differ".)
2. **Mock is a runtime interpreter, not a compiled RT family.** Unlike isType/JSON/binary, `createMockType` has no cache tag and no Go emitter (¬ß3 #1c). It walks the cache at call time. Deliberate ‚Äî mock output is random/non-deterministic and cheap to interpret; compiling it would bloat the cache for a dev-only capability.
3. **`toJsCode` became the cache-module renderer.** mion's "reflection‚ÜíJS source" JIT (toJsCode.ts) is exactly what Go's `module.go` does ahead-of-time. The capability is *more* central in ts-go (it's the whole virtual-module emission), just relocated and not exposed as a user fn.
4. **`equalsHelpers` / `microbenchs` / `xyz-Template` are not runtime features.** A test helper, a benchmark set, and a compiler scaffold respectively ‚Äî correctly absent from the shipped runtime (the test helper IS ported, to `test/util/`).
5. **Per-family factories, no "all fns for T" bundler.** Neither mion (`createRunTypeFunctions.ts`) nor ts-go (`createRTFunctions.ts`) bundles all RT fns for a `T` into one object; both expose `createIsType`/`createGetTypeErrors`/‚Ä¶ individually. (¬ß3 #4.)

## 5. Genuinely-forgotten or partial items, severity-ranked

The sweep found **no entirely-forgotten runtime RT family**. The partial items are:

1. **(Low ‚Äî partial, ¬ß3 #5) `RunType` reflection helpers exposed as data, not methods; `getFamily`/`getJitHash`/`createJitFunction`/`getJitConstants` dropped.** Evidence: mion `types.ts:38-49` methods vs ts-go flat `RunType` (`runtypes/types.ts:62-101`).
