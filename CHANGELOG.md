## [0.10.0] - 2026-07-18

This release rebuilds unknown-key handling around a non-mutating clone and makes the
circular-reference guard pay-for-use. The mutating strip families are removed
(breaking): `createCloneExactShapeFn` returns a clone of the declared shape — unknown
keys dropped by construction, the input never mutated, class prototypes and runtime
types preserved — and `createHasUnknownKeysFn` gains a `{runsAfterValidation: true}`
count fast path. `rejectCircularRefs` moves from a global runtime flag to a
compile-time option (breaking: `setRejectCircularRefs` is removed) that forks the
compiled entry and inlines a guard over a baked cycle-edge skeleton, so a plain
cyclable type ships nothing. Built-in `rt::` pure functions are delivered on demand
instead of riding every bundle, the pure-fn cache honors `emitMode`, wrapper
frameworks get an anonymous pure-fn registration lane plus a structured build report
of every extracted pure fn (the cross-bundle transport surface), and lint setup
collapses to zero config under a single `genDir` convention.

### Features

- **unknown-keys:** [**breaking**] `createCloneExactShapeFn` replaces the mutating
  strip family — `createStripUnknownKeys`, `createUnknownKeysToUndefined`, their
  override twins and Fn types are removed. The clone drops unknown keys by
  construction, never mutates the input (frozen inputs work), and preserves class
  prototypes and runtime types; delete-based stripping measured 3–24x slower and
  flipped objects into V8 dictionary mode. Object-bearing unions fail the build
  (CES001) instead of silently keeping unknown keys.
- **unknown-keys:** [**breaking**] `createHasUnknownKeysFn` gains a compile-time
  `{runsAfterValidation: true}` option: all-required object shapes swap the per-key
  scan for a single enumerable-key count compare (~3x on a 7-prop shape, ~44x at 30
  flat props). Its plain cache-key prefix moved (`trR` → `lRN`) — consumers pinning
  fn-hash prefixes re-pin `huk` once.
- **unknown-keys:** Declared members never drop from the clone: a declared function /
  symbol / Promise-valued property is kept and shared by reference (CES010 / CES015),
  and only undeclared keys strip.
- **core:** [**breaking**] `rejectCircularRefs` becomes a compile-time option on the
  four guarded families (validate, validationErrors, toBinary, jsonEncoder) — the
  armed factory is its own compiled entry with the cycle guard inlined over a baked
  path skeleton (DAGs pass, true cycles flag), and a plain cyclable type ships
  neither the walker nor its RunType bundle. The global `setRejectCircularRefs` /
  `isRejectCircularRefsEnabled` runtime toggles are removed.
- **core:** Built-in `rt::` / `rtFormats::` pure functions are delivered on demand
  through each entry's soft deps instead of shipping in every bundle; the published
  dist hollows the registration bodies (disk cache format v15).
- **core:** The pure-fn cache honors `emitMode` (`code` | `functions` | `both`), so a
  `code`-mode build no longer ships every pure-fn body twice.
- **pure-fn:** Symmetric factory / direct registration (two markers, four
  registrars), plus a wrappable anonymous registration lane with runtime-key lookup
  for wrapper frameworks.
- **pure-fn:** Structured build report of every generated pure fn — call-site span,
  callee attribution resolved through wrappers (`calleeName` / `calleeModule`),
  lane / form, registry key, and the self-contained `paramNames` + `code` payload
  (shape identical across every `moduleMode`). One tri-state `pureFnReport` plugin
  option selects delivery: `'file'` writes `<genDir>/types/pure-fns-report.json` on
  every generate (plugin-free and CLI batch consumers included), `'callback'` fires
  the `onPureFnReport` handler on every unplugin adapter with build + update phases.
  This is the cross-bundle transport surface: host tooling (mion's `serverMapFrom`)
  harvests client-authored mappers from the report and bakes them into the server
  bundle at build time — the wire carries only the content key.

### Bug Fixes

- **core:** Make findCycle's walk state per-call — re-entrancy-safe, no GC pin.
- **core:** Exempt built-in pure-fn soft deps from the JCP001 composite assert.
- **unknown-keys:** Keep absent optional tuple slots absent in the clone.
- **codegen:** Fix the gen-fn-hashes path and complete the devtools sub-kind +
  ReflectionKind mirrors.
- **pure-fn:** The anonymous-lane extractor discovers the marker positions on the
  resolved signature instead of assuming slots 0/1, so a wrapper with leading
  non-marker parameters (`serverMapFrom(source, mapper, hash?)`) extracts: the
  inline-fn pre-filter scans every argument, the injected hash splices at its
  declared slot (skipped optional gaps pad with `undefined`), and overloaded
  wrappers resolve per call site — a marker-free string overload (a name-based
  fallback lane) never extracts.

### Performance

- **core:** findCycle memoizes fully-explored subtrees (O(V+E), not exponential),
  hoists its setup into the factory closure, and uses an array for the descent
  stack.

### Refactoring

- Rename the internal render scheme `virtual:rt/` to `rtmod:/`.
- **config+lint:** One `genDir` convention, per-family lint rules, zero-config
  linting.
- Move the Go-generated TS mirrors into a `go-generated/` dir per package.
- **core:** Rename `rt::findCycleParent` to `rt::findCycle`; delete dead built-in
  pure fns.
- **cachegen:** Funnel every pure-fn reference through `ctx.UsePureFn` with a
  recording tripwire.

### Documentation

- Update virtual-module leftovers to files-mode reality.
- **website:** Reword the pure-fn marker cards away from "inline".

### Testing

- **cloning:** Dedicated suite in the canonical case-file architecture — the full
  serialization case matrix, class-instance and Temporal identity pins, and a fuzzer
  checking the clone against a reference-interpreter oracle.
- **core:** Mirror the DAG-memo and re-entrancy guard cases into serialization.
- **pure-fn:** Runtime and node_modules e2e for the anonymous lane through a
  wrapper.
- **pure-fn:** Build-report attribution + update-lane coverage across both
  moduleModes; leading-param wrapper parity, overload no-extract, and
  optional-gap padding pins (Go + node_modules FE).

## [0.9.3] - 2026-07-15

This release makes the resolver's scan diagnostics robust for source-first consumers and
continues hardening the release pipeline. Scan diagnostics are now scoped to first-party
files, so a dependency resolved to its own TypeScript source (via its `source` export
condition, or a consumer's `customConditions: ["source"]`) never trips the build on the
library's own internal definitions. It generalizes the 0.9.2 marker-only exemption using
TypeScript's own module-resolution provenance, and a source-first consumer now guards it
in the pre-publish matrix. Function cache-key prefixes become version independent, with a
public `getFnHash` derivation, and the website deploy and garble cache gain further guards.

### Features

- **fnhash:** Version-independent fn cache-key prefixes + getFnHash derivation.
- **release:** Guard website deploy against a non-live npm version.

### Bug Fixes

- **resolver:** Scope scan diagnostics to first-party files — a dependency resolved to
  its own `.ts` source no longer false-positives on the library's internal generics.
- **ci:** Key the garble cache by arch so arm64 doesn't run the x64 binary.
- **website:** Don't halt suite-data export on the suite's negative fixtures.

### Refactoring

- **rtx:** Run codegen generators directly, drop the `gen:*` scripts.

### Documentation

- **website:** Drop consumer/table framing from the getFnHash guide.

### Testing

### CI

## [0.9.2] - 2026-07-15

This release continues hardening RunTypes for wrapper frameworks such as mion and
greens the end-to-end release pipeline. Serialization gains a runtime enumerability
guard, so global-inherited and `@nonEnumerable` properties stay off the wire while the
`DataOnly` projection stays sound; class serializers now register once for every generic
instantiation; and mock data and the JSON compact strategy close several soundness gaps.
The bundler plugin fails a lane on error-severity diagnostics by default, and a new
scanner diagnostic (MKR007) flags marker types silently degraded to `any` by unresolved
imports. The seven-platform release gate and pre-publish e2e are green.

### Features

- **serialize:** Runtime enumerability guard for global-inherited and `@nonEnumerable`
  properties, applied only when the property is optional so the `DataOnly` projection
  stays sound.
- **class-serializers:** One registration now covers every generic instantiation.
- **devtools:** Error-severity diagnostics fail every lane (`failOnError`, default on).
- **formats:** Definition-time `mockSample` validation (FMT003 bounds, FMT004
  fail-closed).
- **scanner:** MKR007 diagnoses marker types degraded to `any` by unresolved imports.
- **typeid:** Tuple labels and function parameter names fold into structural type ids.
- **reflection:** Export `RunTypeSubKind` (and its name / value types) from the package
  index.
- **lint:** NE001 flags `@nonEnumerable` on a required property.

- **release:** Green the release gate — unblock the three red jobs (website-build, e2e

## [0.9.0] - 2026-07-09

First public release. RunTypes reads your TypeScript types at build time, through a
native Go binary that queries the real typescript-go (tsgo) checker, and generates
the runtime artifacts TypeScript normally throws away: validators, JSON and binary
codecs, mock data, and reflection. Your types are the schema, so there is no separate
schema language to learn or keep in sync.

### Features

- **Type-driven validation.** `createValidateFn<T>()` emits a specialized type guard and
  `createGetValidationErrorsFn<T>()` a detailed error reporter, both generated ahead of
  time from your type with no runtime reflection and no first-call cost.
- **JSON serialization.** `createJsonEncoderFn<T>()` / `createJsonDecoderFn<T>()`, with
  `clone` (default), `mutate`, and `direct` strategies.
- **Binary serialization.** `createBinaryEncoderFn<T>()` / `createBinaryDecoderFn<T>()` emit
  a compact binary format.
- **Type formats.** Format-constrained types (`int8`, `uint16`, currency, `min` / `max`
  bounds, and more) tighten validation and pack values into far fewer binary bytes than
  an unconstrained `number` or `bigint`.
- **Reflection and stable type ids.** `getRunTypeId` in both a static
  `getRunTypeId<T>()` form and a value-first `getRunTypeId(value)` form; every type
  collapses to one stable structural id, so identical shapes share a single cache entry.
- **Schema builders.** `RT.*` builders are a second door into the same engine — a plain
  type and an `RT.*` schema resolve to the identical representation — with `InferType<>` to
  extract the type back out.
- **Mock data.** `createMockDataFn<T>()` produces realistic sample values, drawing on the
  committed `MockData<T>` enrichment maps.
- **AI-assisted enrichment.** Committed, type-keyed `FriendlyText<T>` (human labels and
  friendly validation messages, with per-locale i18n) and `MockData<T>` maps, maintained
  by the `ts-runtypes` CLI (`describe` / `gen` / `check`) and the enrich skill, and
  rendered at runtime by `createFriendlyText<T>()` / `createFriendlyTextI18n()`.
- **Custom per-type overrides.** `overrideValidate`, `overrideJsonEncoder`,
  `overrideBinaryEncoder`, and the rest register a hand-tuned pure function for one type;
  the override folds into the type id and propagates to every type that contains it.
- **A serializable-data contract.** Validators and codecs operate on the serializable
  projection `DataOnly<T>`; non-serializable members (functions, methods, symbol keys)
  are dropped with a build-time warning, and decoders return `DataOnly<T>`.
- **Cross-bundler plugin.** `@ts-runtypes/devtools` works across Vite, Rollup, webpack,
  Rspack, and esbuild (unplugin), carries zero runtime dependencies, and writes each
  cache entry as its own real module so bundlers tree-shake and code-split natively.
  Options cover emit mode, module grouping, and the transform wire mode.
- **tsc-like compile CLI.** `ts-runtypes --compile` is a batch build for hosts without a
  bundler plugin; it emits `.js` via tsgo and composes source maps back to the original
  TypeScript.
- **Editor and CI linting.** The `@ts-runtypes/devtools/eslint` plugin (OXlint /
  ESLint v9) surfaces every compiler diagnostic in the editor and gates enrichment-file
  hygiene at commit time and in CI.
- **Automatic binary distribution.** `@ts-runtypes/bin` resolves the prebuilt resolver
  binary for the host from per-platform optional dependencies — no postinstall
  downloader.
- **Single configuration surface.** Project options live in the `ts-runtypes` entry
  under `compilerOptions.plugins` in `tsconfig.json`, with tsc-style precedence, read
  identically by every host.

### Documentation

- Full documentation site at https://runtypes.pages.dev — introduction, guides
  (validation, serialization, formats, reflection, mocking, pure functions,
  configuration, linting), AI-integration guides, and generated suites and benchmarks.
- README reworked to a concise project overview (RunTypes / Why / Use).
