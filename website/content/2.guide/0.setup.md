## Install

You need two things: the runtime package (the markers and the `createX` factories) and the Vite plugin (it reads your types at build time and writes the functions back).

::code-group
```bash [pnpm]
pnpm add @mionjs/ts-go-run-types vite-plugin-runtypes
```
```bash [npm]
npm install @mionjs/ts-go-run-types vite-plugin-runtypes
```
```bash [yarn]
yarn add @mionjs/ts-go-run-types vite-plugin-runtypes
```
::

::note
The plugin talks to a small Go binary that reads your types (TypeScript 7 ships its compiler as Go, and the old transformer hook is gone — so we go in through a native side-channel). Point `binary` at it. The full option list lives in the [Plugin & CLI reference](/reference/plugin-and-cli).
::

## Turn on type metadata

ts-run-types reads your types, so the type-checker needs to keep the metadata around. Flip `"reflection": true` in your `tsconfig.json`:

```json [tsconfig.json]
{
  "compilerOptions": {
    "reflection": true
  }
}
```

That's the whole setup. No decorators, no codegen step you run by hand, no schema files.

## Hello, validator

Sanity check: write a plain type, ask for a validator, run it.

<code-import path="packages/examples/src/guide/setup-hello-validator.ts" lang="ts" />

If `isUser` returns `true` then `false`, you're done — the build generated a real function from your type. TS erased your `User`; we put it back.

::tip
Nothing happening? The factories fall back to a no-op when the plugin isn't running, so a `createValidate` that always returns `true` usually means the plugin didn't load or `reflection` is off. Double-check both, then restart the dev server.
::
