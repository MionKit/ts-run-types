Every `createX<T>()` factory turns a type into a purpose-built function the build writes out for you — no schema object, no runtime reflection. They all share one calling convention and one contract, so once you've seen one you've seen them all.

This page covers **validation**: the type guards, the error reports, and unknown-key handling. ([Serialization](/guide/serialization), [mocking](/guide/mocking) and [reflection](/guide/markers-and-reflection) get their own pages.)

## Three ways to call any factory

Every factory takes your type three different ways. They all resolve to the same generated function — use whichever fits the call site.

<code-import path="packages/examples/src/guide/validation-three-forms.ts" lang="ts" commentStart="// start-forms" commentEnd="// end-forms" />

- **Type-first** — you supply `<T>`, no value needed.
- **Value-first** — `T` is inferred from a value you already have.
- **Schema-first** — pass an `RT.*` schema; `T` comes from the schema.

The rest of these guide pages use whichever form reads best for the example — but every one of them accepts all three.

## Tuning with `ValidateOptions`

Both validators take an options object as a **literal at the call site**. The build reads the literal and routes you to a specialized variant — nothing is parsed at runtime.

<code-import path="packages/examples/src/guide/validation-options.ts" lang="ts" commentStart="// start-options" commentEnd="// end-options" />

| Option | What it does |
| --- | --- |
| `noLiterals` | Literal checks degrade to their base type — `literal 'a'` accepts any string, `literal 2` accepts any finite number. |
| `noIsArrayCheck` | Skips the leading `Array.isArray()` guard on array validators, for when you've already proven the value is an array upstream. |

::warning
Pass the options as an **object literal** at the call site. It's read at build time, so a variable won't work — the build can't see its value.
::

## Unknown keys

Four tools for properties that aren't in your declared type. These check for *extra* keys — to check that the *declared* keys hold the *right types*, that's validation above.

| Factory | What it does |
| --- | --- |
| `createHasUnknownKeys` | Predicate — `true` if the value carries any undeclared key. |
| `createStripUnknownKeys` | Deletes undeclared keys in place, returns the same reference. |
| `createUnknownKeyErrors` | One `{path, expected: 'never'}` per undeclared key — same shape as `createGetValidationErrors`. |
| `createUnknownKeysToUndefined` | Keeps the key but sets it to `undefined` instead of deleting it. |
