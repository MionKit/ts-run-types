The pure type is the fast path — there's nothing to write but the type you already have. The schema is a real value you can store in a variable, pass to a function, or build up piece by piece.

## Get the type back with `Static`

A schema is a value, but sometimes you want the TypeScript type it stands for. `Static<typeof schema>` hands it back:

So you never write a shape twice. Start from a type and reflect it, or start from a schema and `Static` your way to the type — either direction, one source of truth.

## Mix them freely

They're the same thing, so you can use both in the same file — a plain type here, a schema there, even nest one kind of thinking inside the other.

<code-import path="packages/examples/src/guide/types-vs-schemas-mixed.ts" lang="ts" />
