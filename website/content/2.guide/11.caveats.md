## Type-only imports erase the call

`import type { … }` tells the compiler "this is types only, drop it from the output." Perfect for a `Format*` alias. **Not** perfect for a factory — a type-only import of `createValidate` erases the function, so the call disappears and nothing gets generated.

<code-import path="packages/examples/src/guide/caveats-type-only-import.ts" lang="ts" />

Rule of thumb: import the **factories as values**, import the **format/marker types** however you like.

## Missing type annotations

The build can only read a type that's actually there. Hand `getRunTypeId` a value with a precise static type and you get a precise id. Hand it an `any` — from `JSON.parse`, an untyped library, an `as any` — and there's nothing to read.

<code-import path="packages/examples/src/guide/caveats-missing-annotation.ts" lang="ts" />

If reflection or validation is mysteriously vague, check whether the type got lost upstream. A quick `as User` at the boundary fixes it.

## `typeof` — the type operator, not the runtime one

To go from a value to its id with the static `getRunTypeId<T>()`, use TypeScript's **`typeof` type operator**. The runtime `typeof` is a different beast — it returns a string like `'object'`, which is a value, not a type, and means nothing to ts-run-types.

<code-import path="packages/examples/src/guide/caveats-typeof-runtime.ts" lang="ts" />

Easiest path from a value: just use `getRunTypeId(value)` and skip the `typeof` dance entirely.

## Unions are precise; `any` is not

Unions are read member by member — a discriminated union validates exactly the member that matches, and rejects the rest. That part just works.

`any` is the opposite of a type. `createValidate<any>()` is a **noop** — it accepts everything and emits a build-time diagnostic to say so. Same for serializers built on `any`: best-effort, with a warning.

<code-import path="packages/examples/src/guide/caveats-unions-and-any.ts" lang="ts" />

::warning
A validator that never returns `false` almost always means a stray `any` (or an `unknown` you forgot to narrow) reached the type. The build-time diagnostic is your tip-off — don't ignore it.
::
