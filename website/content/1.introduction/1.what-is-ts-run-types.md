# What is ts-run-types

TypeScript erases your types and calls it a feature. At runtime your carefully-typed `User` is just an object — nobody's checking anything.

**ts-run-types** reads your types at build time and hands the runtime back what it lost: validators, JSON + binary (de)serializers, mock-data generators and real reflection. We respectfully bolted the runtime back on.

## The 60-second version

The compiler decided it's "just a linter" and throws your types away before your code ever runs. So we read them first.

A build-time step looks at your TypeScript types and generates a specialized function for each thing you ask for. You write a normal type, you call `createValidate<User>()`, and you get a real validator — no schema, no decorators, no writing your shapes twice.

It runs at build time, not run time. There's no reflection happening when your app is live, and the Vite plugin adds **zero** runtime dependencies.

::note
Under the hood, TypeScript 7 ships the compiler as a Go binary and the old transformer hook is gone — so we read your types through a native Go side-channel instead. You never touch it. [Here's how that works.](/how-it-works/the-side-channel)
::

## What you get

One type, the whole toolbelt — all generated from the same shared understanding of your shape, so the validator and the serializer never disagree:

- **Validation** — `createValidate` for fast yes/no checks, `createGetValidationErrors` for detailed reports.
- **JSON that round-trips** — `createJsonEncoder` / `createJsonDecoder`. `Date`, `BigInt`, `Map`, `Set` survive the trip.
- **Binary codec** — `createBinaryEncoder` / `createBinaryDecoder` for compact, fast payloads.
- **Mock data** — `createMockType` invents valid, type-shaped data for tests and fixtures.
- **Reflection** — `getRunTypeId` / `reflectRunTypeId`: a stable id for any type, from the type or from a value.
- **Type formats** — `email`, `uuidv4`, `ipv4`, `int32`, `positive`… baked straight into your types.

And because every generated function is its own module and the caches only hold what you actually call, you ship only what you use. A file that just reflects an id ships zero validation code.
