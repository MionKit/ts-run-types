## Requirements

- **TypeScript 7 (tsgo).** This is the interesting bit. TypeScript 7 ships the compiler as a Go binary, and the old transformer API — the hook most reflection libraries plugged into — is gone. So instead of a transformer, ts-run-types reads your types through a native Go side-channel that talks to tsgo's checker directly. You don't interact with it; you just install the plugin. If you're curious, [The Side-Channel](/how-it-works/the-side-channel) explains the whole trick.
- **Vite.** The current integration is the Vite plugin. It runs the type-reading at build time and rewrites your `createX<T>()` calls in place.
- **A bundler that tree-shakes** (Vite already does). Every generated function is its own module, so you only ship the ones you actually call.

::note
No runtime reflection, no decorators, no `reflect-metadata`. The type reading happens entirely at build time. When your app is live, there's nothing extra running.
::

## The two packages

You install two things, and they have very different jobs.

- **`@mionjs/ts-go-run-types`** — the runtime package. It carries the markers (`InjectRunTypeId<T>`), the `getRunTypeId` / `reflectRunTypeId` reflection helpers, the `createX<T>()` factory signatures, and the `RT.*` schema builders. This is the only one that ends up in your bundle, and it ships **zero** dependencies.
- **`vite-plugin-runtypes`** — the build-time plugin (a `devDependency`). It spawns the Go binary, reads your types, rewrites your calls, and serves the generated code. It contributes nothing to your production runtime.

## tsconfig

Turn on `reflection` so the type information is available to the plugin:

```json [tsconfig.json]
{
  "compilerOptions": {
    "reflection": true
  }
}
```

::warning
This flag is not optional. Without it the build has nothing to read, and your `createX<T>()` calls won't generate anything. If something "just isn't working," check this first.
::
