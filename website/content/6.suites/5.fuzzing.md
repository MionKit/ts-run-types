The suites on the previous pages are hand-written: real types, real values, known answers. They're thorough — but they only try the inputs **we** thought of. Fuzzing covers the rest: the weird, the malformed, and the cases nobody would write by hand.

## What it is

Fuzzing throws a flood of **randomly generated inputs** at the runtime functions and checks that a handful of rules always hold — no matter how strange the input. Instead of asking "does `validate` return `true` for this hand-picked value?", it asks "does `validate` ever misbehave, across thousands of random values?"

Every run is **seeded**: a single number reproduces the whole sequence, so any failure replays exactly for debugging.

## Why we do it

Hand-written tests pass because they check what we expected. Real-world data — and real-world *types* — are messier. Fuzzing is how we close the gap between "works on our examples" and "works on anything."

It earns its keep: the very first run surfaced a real bug — a binary encoder that could overflow its buffer on a perfectly valid value after a run of smaller ones. Found, fixed, and pinned with a regression test.

## How we use it

We fuzz on two levels:

- **Random data** — for a fixed set of types, generate thousands of valid values, deliberately-broken values, and pure junk, and check the rules against every one.
- **Random types** — generate *the types themselves*: arbitrary, valid-but-weird TypeScript (unions, intersections, recursive interfaces, classes, index signatures, `Map` / `Set`, non-serializable shapes…), run the whole build pipeline, and confirm every generated validator and (de)serializer behaves.

The rules are properties that must hold for **all** inputs — never a hand-written expected answer. For example:

- `validate` always returns a boolean and never throws, on *any* input.
- `validate(x)` is `true` exactly when `getValidationErrors(x)` is empty — the two never disagree.
- A value survives a JSON or binary round-trip unchanged.

Break any rule and the seed that broke it is reported, ready to replay.
