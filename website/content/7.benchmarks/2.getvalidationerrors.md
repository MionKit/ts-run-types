## Validation errors — `getValidationErrors`

The full error report — walk the value and collect every problem. Strictly more work than the boolean [is-valid check](/benchmarks/validation). Higher is better; `—` = no result, `FAIL` = wrong answer, `n-a` = unsupported. Hover any row for each competitor's source.

::note
AJV and zod always compute errors (their validator _is_ the error path), so these match their validate speed; ts-run-types and TypeBox pay extra to build the report.
::
