ts-run-types surfaces problems at build time, formatted like `tsc` so your editor's problem matcher picks them up. The Go binary ships only the code (plus a few positional args); the wording is resolved from a catalog. The clean line:

::note
**Warning** = an expected, harmless drop (e.g. a function-typed property a serializer can't carry) — your build keeps going. **Error** = the generated function would throw at runtime, so the build is halted.
::

Every diagnostic carries a single-line headline and a longer detail block with a concrete fix (shown in your IDE hover / verbose log).

## Marker & compile-time args (MKR / CTA / PFN)

These fire when a marker call site can't be resolved at build time.

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `MKR001` | Warning | A reflect-form marker (`reflectRunTypeId(value)`) runs its argument at runtime just to read its type. Use the static form `createValidate<ReturnType<typeof fn>>()` or pass an existing value. |
| `MKR003` | Error | Marker call sits inside a generic function — `T` is unresolved, so no id can be computed. Inline at each concrete call site, or accept a pre-computed `InjectRunTypeId<T>` from the caller. |
| `CTA001` | Error | `CompTimeArgs<T>` argument must be a literal (or a module-scope `const` whose initializer is fully literal). Inline it or move it to a literal const. |
| `CTA002` | Error | `CompTimeArgs<T>` literal nesting exceeds the depth cap (16). Flatten or split across smaller args. |
| `CTA003` | Error | `CompTimeArgs<T>` literal contains a forbidden construct (spread, computed key, call, ternary, template substitution). Replace with its literal value. |
| `PFN001` | Error | `PureFunction<F>` argument must be an inline arrow / function expression (or a const initialized to one). Inline the function. |

## Pure functions (PFE)

Purity rules for `registerPureFnFactory` bodies — these run at compile time, so the body must be self-contained and synchronous. (`PFE9001`–`PFE9003` were retired; shape errors now flow through `CTA001` / `PFN001`.)

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `PFE9004` | Error | Duplicate registration for the same `ns::fn` key with a different body. Make them identical or keep one. |
| `PFE9005` | Error | Factory uses destructured params — use a single identifier and destructure inside the body. |
| `PFE9006` | Error | `this` is not allowed in a pure-fn body. Pass an explicit parameter instead. |
| `PFE9007` | Error | `async`/`await` is not allowed — pure fns must be synchronous. |
| `PFE9008` | Error | `yield` / generators are not allowed. Return an array / iterable instead. |
| `PFE9009` | Error | `import()` is not allowed. Use a top-level `import` or pass the module in. |
| `PFE9010` | Error | A forbidden global (`eval`, `Function`, `fetch`, `process`, `globalThis`, …) is referenced. Remove it or pass it in. |
| `PFE9011` | Error | An outer-scope variable is captured — pure fns can't reach outside their body. Pass it in as a parameter. |
| `PFE9012` | Error | A pure fn is referenced by an RT function but never registered. Call `registerPureFnFactory(ns, fn, …)` and make sure the file is scanned. |
| `PFE9013` | Error | A `usePureFn` / `getPureFn` key must be a string literal or same-scope `const` string. |

## Validation root throws (VL / VE)

At a **root** position a non-serializable type can't be validated meaningfully, so the factory throws.

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `VL001` / `VE001` | Error | Root is a non-serializable built-in (`Map`, `Set`, typed array, `Promise`, …). Convert to a plain object/array. |
| `VL002` / `VE002` | Error | Root is a `symbol` — symbols can't round-trip. Use a stable string key (often a literal union). |

## Validation noop warnings (VL021 / VE020)

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `VL021` | Warning | `validate` on `any` / `unknown` always returns `true` — nothing is checked. Narrow to the real shape. |
| `VE020` | Warning | `validationErrors` on `any` / `unknown` always returns an empty array. Narrow to the real shape. |

## Serialization root throws

At a root or propagating position (array element, tuple slot, union member, function param/return) a non-serializable member makes the encoder/decoder throw. One family per operation: **PJ** `prepareForJson`, **PJS** `prepareForJsonSafe`, **RJ** `restoreFromJson`, **SJ** `stringifyJson`, **TB** `toBinary`, **FB** `fromBinary`. The trailing digit names the cause:

| Suffix | Cause |
|--------|-------|
| `*001` | `never` — the empty type has no value to encode. |
| `*002` | Non-serializable built-in (`Map`/`Set`/typed array/`Promise`/…). |
| `*003` | A `function` — no value form to serialize. |
| `*004` | Array of un-serializable elements (dropping elements would change the length). |
| `*005` | `symbol` (for JSON families; for **TB**/**FB** `*005` is a second non-serializable case and `*006` is `symbol`). |

So `PJ002` = "cannot encode `Map` to JSON", `TB006` = "cannot serialise `symbol` to binary", `FB001` = "cannot deserialise `never` from binary", and so on across all six families.

## Member-drop warnings (`*010`–`*013`)

At a **property** position, a non-serializable member is silently dropped (your build keeps going). These exist across every family: validate (`VL`), validationErrors (`VE`), the six serialization families (`PJ`/`PJS`/`RJ`/`SJ`/`TB`/`FB`), and the unknown-key families (`HUK`/`SUK`/`UKE`/`UKU`/`UKW`).

| Suffix | What's dropped |
|--------|----------------|
| `*010` | A function-typed property. |
| `*011` | A method. |
| `*012` | A static member. |
| `*013` | A symbol-keyed property. |

::note
These are **by design** — see the "validate contract" in the architecture docs. `interface User { name: string; onClick: () => void }` yields a validator that only checks `name`; the `VL010` warning is the only signal, and it is *not* an error.
::

## Class serializer (CLS)

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `CLS001` | Warning | A user class is serialized structurally (decoded to a plain object — `instanceof` is false, methods are gone). Register a `registerClassSerializer(name, …)` if you need a real instance back. |

## Type formats (FMT)

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `FMT001` | Error | A format's `mockSample` doesn't match its own pattern — mocking would produce an invalid value. Fix the sample or the pattern. |
| `FMT002` | Error | Invalid type-format params. The detail names the offending parameter. |

## Temporal (TMP)

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `TMP001` | Warning | A `Temporal.*` type resolved to `any` because the Temporal lib isn't in your tsconfig — the validator would accept anything. Add `"ESNext.Temporal"` to `compilerOptions.lib`. |

## Internal (JCP)

| Code | Severity | Meaning / fix |
|------|----------|----------------|
| `JCP001` | Error | Internal invariant breach — a JSON composite references a primitive entry that never rendered. Not a user error; please file an issue. |
