# Unknown Keys

Sometimes a value arrives with extra keys your type never declared — a sneaky `admin: true`, a leftover `token`. Four small functions deal with them, each generated from your type. Pick the one that matches what you want to do about it.

## Detect them — `createHasUnknownKeys`

A quick predicate. `true` if the value carries any key the type didn't declare.

<code-import path="packages/examples/src/guide/unknown-keys-has.ts" lang="ts" />

## Remove them — `createStripUnknownKeys`

Deletes the undeclared keys in place and returns the same reference. Good for sanitizing an object before you store or forward it.

<code-import path="packages/examples/src/guide/unknown-keys-strip.ts" lang="ts" />

## Report them — `createUnknownKeyErrors`

One `{path, expected: 'never'}` entry per undeclared key — same error shape as `createGetValidationErrors`, so you can fold it into the same reporting.

<code-import path="packages/examples/src/guide/unknown-keys-errors.ts" lang="ts" />

## Blank them out — `createUnknownKeysToUndefined`

Like strip, but it keeps the key and sets its value to `undefined` instead of deleting it. Handy when downstream code expects the key to exist.

<code-import path="packages/examples/src/guide/unknown-keys-to-undefined.ts" lang="ts" />

::note
These check for keys outside your declared shape. For checking that the *declared* keys hold the *right types*, that's [Validation](/guide/validation).
::
