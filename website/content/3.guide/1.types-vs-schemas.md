# Types vs Schemas

There are two ways to tell ts-run-types about a shape. A plain TypeScript type, or the `RT.*` schema builders. They compile to the exact same thing, so this is purely about taste.

The pure type is the fast path — there's nothing to write but the type you already have. The schema is a real value you can store in a variable, pass to a function, or build up piece by piece.

## When to use which

::card-group
::card{title="Reach for a pure type" icon="i-lucide-type"}
You already have the type. You want zero ceremony. You're describing a static shape and never need a runtime handle to it. This is most of the time.
::
::card{title="Reach for a schema" icon="i-lucide-blocks"}
You like the Zod / TypeBox builder feel. You want a value to pass around or compose. You're building shapes dynamically, or sharing one definition across modules.
::
::

Either way you end up at the same validator, the same JSON codec, the same mock. Pick whichever reads better to you.
