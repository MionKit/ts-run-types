# Playground

Write a TypeScript type on the left and see the **RunType graph** RunTypes
resolves for it on the right. The whole resolver (Go + the TypeScript compiler)
runs in your browser as a WebAssembly module, so nothing is sent to a server.

This is the same reflection data RunTypes uses to generate validators,
serializers, and mock data at build time. Try changing the type: add a union, a
nested object, an optional property, or a literal, and watch the graph update.

::runtypes-playground
::

::note
The first resolve loads the WebAssembly module (a few megabytes), so it may take
a moment. After that, edits resolve instantly. This is a proof of concept built
on the same compiler the library uses during your build.
::
