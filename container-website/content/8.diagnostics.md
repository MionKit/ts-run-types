# Build-time diagnostics

RunTypes resolves your types and writes the matching functions while your project builds, so it can flag problems before any code runs. Each finding prints a short code, like `VL010` or `PJ001`, with a one line message. Search the full catalog below by code, or by words from the message.

::note
**Warning** is an expected, safe drop, and the build keeps going. RunTypes works on the data in your type, the part that survives JSON, so anything with no data form (a method, a function valued property, a symbol key) is left out of the generated function. The code names what was left out, so it is never a surprise.
::

::note
**Error** means the generated function would throw the moment you call it, so the build stops. This happens when a whole value has no data form, like a type that resolves to `never`, a bare `symbol`, or a `Map` at the position being serialized. Change the type, or pick a shape that can be represented as data.
::

The letters tell you which feature produced the diagnostic (`VL` is `createValidate`, `PJ` is `prepareForJson`, `MKR` is the type markers). The number tells you where it sits: `001` through `009` are root-position errors, where the whole type has no data form, and `010` and up are member-level warnings, the silent drops made visible.
